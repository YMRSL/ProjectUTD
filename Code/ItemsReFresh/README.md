# ItemsRefresh 插件

ItemsRefresh 是一个专为 Minecraft Mohist 服务端设计的插件，用于管理和刷新选定区域内容器中的物品。

## 功能特性

### 1. 小木斧Mod兼容优化
- 支持"左键选起点+右键选终点"的标准选区逻辑
- 自动检测选区是否合法（如是否超出世界边界、是否为空气区域）
- 兼容小木斧Mod的特殊特性

### 2. 容器检测与管理细化
- 仅检测"已放置且未被破坏"的木箱、展示框、木桶
- 排除"物品形态"的容器（如背包内的木箱）
- 记录容器内初始物品状态
- 被检测到的容器可在管理员视角下显示特殊粒子效果

### 3. 完善的指令系统
- 选区管理：确认选区、清空选区
- 容器管理：选择目标选区内的容器
- 刷新界面：打开区域刷新管理界面
- 数据管理：备份和加载刷新配置
- 日志管理：查看刷新日志

### 4. 区域刷新管理界面
- 27格物品栏，与原版背包布局一致
- 界面数据与选区绑定，不同选区独立配置
- 数据持久化存储，服务器重启后保留配置
- 权限控制，仅管理员可编辑界面

### 5. 随机刻刷新机制
- 参考小麦生长判定逻辑，默认每游戏刻有1/2000概率对单个容器触发随机刻
- 可通过指令调整刷新频率
- 智能刷新规则，避免物品溢出

## 权限节点

- `area.refresh.admin` - 管理员权限，包含所有子权限（默认OP拥有）
- `area.refresh.edit` - 编辑刷新界面权限
- `area.container.select` - 选择容器权限
- `area.refresh.log.view` - 查看刷新日志权限
- `area.refresh.data.manage` - 管理数据备份权限

## 指令使用

### 基础指令
- `/area select confirm` - 确认小木斧所选选区
- `/area select clear` - 清空已绑定的选区

### 容器管理
- `/area container select` - 选中目标选区内所有符合条件的容器

### 刷新管理
- `/area refresh gui open` - 打开区域刷新管理界面
- `/area refresh rate [数值]` - 设置随机刻触发频率（范围1-10000）

### 数据管理
- `/area refresh data save [名称]` - 备份当前选区的刷新配置
- `/area refresh data load [名称]` - 加载指定名称的备份数据

### 日志管理
- `/area refresh log view [条数]` - 查看最近指定条数的刷新日志

## 配置文件

插件的配置文件位于 `plugins/ItemsRefresh/config.yml`，可以调整以下设置：

- `random-tick-frequency` - 随机刻触发频率
- `max-selection-size` - 最大选区范围
- `allowed-containers` - 允许的容器类型
- `particle-effect` - 粒子效果设置
- `logging` - 日志设置
- `data-storage` - 数据存储设置

## 安装说明

1. 将插件的JAR文件放入服务器的 `plugins` 文件夹中
2. 重启服务器以加载插件
3. 插件会自动生成默认配置文件
4. 根据需要调整配置文件中的设置
5. 分配适当的权限给玩家

## 注意事项

- 请确保您的服务器运行的是 Mohist 服务端，以兼容本插件
- 过大的选区可能会导致服务器性能下降，请合理设置选区范围
- 编辑刷新界面时，物品堆叠数量会自动调整为最大堆叠数
- 若选区内的容器被破坏，系统会自动从容器列表中移除该容器

## 兼容性

- Minecraft 版本：1.17.1 及以上
- 服务端类型：Mohist 服务端
- 兼容大多数常见的插件，但不保证与所有插件兼容

## 常见问题

**Q: 为什么我无法选择容器？**
A: 请确认您拥有 `area.container.select` 权限，并且已经成功确认了选区。

**Q: 刷新机制是如何工作的？**
A: 插件会按照配置的频率（默认每游戏刻1/2000概率）随机选择一个容器进行刷新，根据刷新界面中的配置添加物品。

**Q: 如何备份我的刷新配置？**
A: 使用 `/area refresh data save [名称]` 命令可以备份当前选区的刷新配置，使用 `/area refresh data load [名称]` 命令可以加载备份。
