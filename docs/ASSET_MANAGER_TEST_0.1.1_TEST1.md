# UTD Asset Manager 0.1.1-test1 实机测试清单

测试目标：先确认 O 键界面恢复正常，再验证人工标注、稳定变体、状态反馈、持久化和导出。

## 测试版本

- 客户端：`1.21.1-NeoForge_21.1.233`
- Mod：`utd_asset_manager-0.1.1-test1.jar`
- SHA-256：`396779EC9111646099827C64F3786409B79EC883F15709CA971387A265C7D3A0`
- 旧 `0.1.0-mvp` 已移入本地回滚目录，没有删除。
- 不需要清空现有 `config/utd_asset_manager/whitelist.json`。

## P0：先测界面修复（约 3 分钟）

- [ ] **A1 版本确认**：启动游戏后，在 Mod 列表或 `latest.log` 中确认加载的是 `UTD Asset Manager 0.1.1-test1`，且没有同时加载 `0.1.0-mvp`。
- [ ] **A2 O 键首开**：进入测试世界并按 `O`。预期三栏界面、标题、列表和检查器全部清晰；不能再出现“主体模糊、只有搜索框清晰”。
- [ ] **A3 基础交互**：输入搜索词、切换状态分组、滚动列表、选择一条记录。预期搜索框与所有按钮可操作，选中项和右侧检查器同步变化。
- [ ] **A4 关闭/返回**：按 `Esc` 关闭界面。预期返回打开前的游戏或容器画面，不崩溃、不出现残留模糊。

若 A1–A4 任一失败，先停止后续测试，保留截图和 `latest.log` 发给 Codex。

## P1：候选范围与人工标注（约 5 分钟）

- [ ] **B1 背包候选**：背包、盔甲栏、副手各放一个不同物品，再按 `O`。预期这些物品出现在候选列表中。
- [ ] **B2 容器候选**：打开一个箱子，在箱子界面仍开着时按 `O`。预期箱子可见槽位中的物品进入候选；按 `Esc` 应返回该箱子界面。
- [ ] **B3 单项标注**：选择一个普通物品并标注。预期 `human_selected` 变为已标注，Tooltip 和界面状态立即更新。
- [ ] **B4 取消与恢复**：取消该物品，再重新标注。预期没有重复记录，计数随操作正确变化。
- [ ] **B5 批量操作**：用搜索或状态筛选出 2–3 项，测试“标注可见/取消可见”。预期只影响当前可见记录。

## P1：TaCZ / FPE 稳定身份（约 5–10 分钟）

- [ ] **C1 TaCZ 枪械**：标注一把 TaCZ 枪，记住选中记录和总标注数；关闭界面，射击或切换射击模式后重新按 `O`。预期仍匹配同一个 `GunId` 资产，不能多出一条重复记录。
- [ ] **C2 TaCZ 区分**：换一把不同 `GunId`、但基础 registry id 相同的枪。预期它能形成另一条独立资产，不会与上一把合并。
- [ ] **C3 FPE 食物（有合适物品时）**：标注一个 FPE pack food，产生食用/耐久状态变化后重新查看。预期仍匹配同一个 `food_id`；不同 `food_id` 不能合并。没有合适物品可记为 `SKIP`。

## P1：状态、重启与导出（约 5 分钟）

- [ ] **D1 状态反馈**：检查一个已有配方或 Loot 的物品。预期 Tooltip/检查器能显示配方输入/产出数量、Loot 启停/等级、同步状态和问题。
- [ ] **D2 状态口径**：出现 `local_only`，或循环涉及物品显示 `error`，目前属于首份内容清单的真实状态，不算本次 UI 测试失败。
- [ ] **D3 持久化**：完全退出并重启客户端，再按 `O`。预期 B/C 阶段保留的人工标注仍存在。
- [ ] **D4 导出**：输入 `/utdasset export`。预期聊天提示导出成功，并在 `config/utd_asset_manager/exports/` 产生新 JSON；中文名、registry id、translation key、asset key 和稳定变体字段存在。
