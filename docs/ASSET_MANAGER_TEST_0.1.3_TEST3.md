# UTD Asset Manager 0.1.3-test3 项目目录测试清单

本轮目标：在不改变 test2 本机标注流程的前提下，把桌面端 `status_manifest.json` 作为只读“项目目录”接入游戏。

## 测试版本

- 客户端：`1.21.1-NeoForge_21.1.233`
- Mod：`utd_asset_manager-0.1.3-test3.jar`
- SHA-256：`07A9EEC286A6D7A7F4050673C394FB66677D2330247A08B1010CC4BA913113D8`

## P0：本机功能回归（约 3 分钟）

- [ ] **L1 默认页**：进入世界按 `O`。预期默认选中“本机”，界面清晰、不虚化。
- [ ] **L2 容器快捷键**：分别在背包和箱子界面按 `O`。预期仍能打开；`Esc` 返回原界面。
- [ ] **L3 本机标注**：在“本机”页标注或取消一件测试物品。预期操作正常，当前 10 条正式标注不丢失。

## P0：项目目录与只读保护（约 5 分钟）

- [ ] **P1 完整目录**：点击中间列表上方“项目目录”。清空搜索并选择“全部”，预期总数为 `851`。
- [ ] **P2 历史标注**：选择“人工标注”，预期总数为 `316`。
- [ ] **P3 已纳管**：选择“已纳管”，预期总数为 `846`。
- [ ] **P4 只读按钮**：选择任意项目记录。预期单项按钮显示“只读”，左侧批量按钮显示不可修改；点击这些区域不会改动本机白名单。
- [ ] **P5 本机数量不变**：切回“本机”，查看顶部“本机标注”计数。若 L3 没有保留额外测试物品，预期仍为 `10`；项目目录的 851 条不能进入本机白名单。注意：“人工标注”筛选也会提示背包中命中过项目历史标注的物品，因此列表可见数不等同于本机白名单总数。
- [ ] **P6 导出隔离**：在项目目录页点击“导出本机”。预期导出成功，导出内容仍只包含本机白名单，不是 851 条项目记录。

## P1：搜索与变体显示（约 3 分钟）

- [ ] **V1 枪械变体搜索**：项目目录选择“全部”，搜索 `GunId=hamster:webley`。预期能找到对应记录，第二行显示完整 `GunId`，图标位置显示“变”占位，不伪装成精确枪械堆栈。
- [ ] **V2 同基础物品多变体**：搜索 `tacz:modern_kinetic_gun`。预期能滚动看到多个不同 `GunId`，不会被合并成一条。
- [ ] **V3 FPE 变体**：搜索 `food_id=firstpersonfoodeating:i_bang_a`。预期能找到对应 Loot 变体并显示“掉”状态。
- [ ] **V4 普通物品**：搜索 `minecraft:iron_axe`。预期普通目录项可显示游戏内基础图标和当前客户端名称；若资源包没有中文翻译，允许保留英文。

## P1：刷新与稳定性（约 2 分钟）

- [ ] **S1 滚动**：在 851 条“全部”列表中连续滚动、点击滚动条和切换筛选。预期无明显卡顿、选中项正常。
- [ ] **S2 重载**：点击“重载”。预期目录数量保持正确，界面不崩溃。
- [ ] **S3 返回本机**：切回“本机”后搜索、标注、导出仍正常。
